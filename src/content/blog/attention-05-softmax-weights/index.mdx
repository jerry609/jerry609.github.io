## 这一讲要回答什么

上一讲我们知道 $QK^\top$ 产生了原始分数矩阵——每一行是一个 Query 对所有 Key 的匹配分数。但这些分数可以是任意实数：正的、负的、大的、小的。

它们不能直接当"注意力权重"用，因为：

- 负分数怎么理解？注意力为 -0.3 是什么意思？
- 分数没有统一量纲，不同行之间无法比较
- 不满足"分配比例"的语义——权重应该加起来等于 1

所以需要一个函数，把任意实数变成**合法的概率分布**。这个函数就是 softmax：

$$
\alpha_j = \text{softmax}(s_j) = \frac{e^{s_j}}{\sum_{k=1}^{n} e^{s_k}}
$$

这一讲要回答：**为什么偏偏是 softmax？它的三个性质为什么恰好满足"注意力权重"的要求？**

## softmax 的三个关键性质

### 性质 1：非负

$$
e^{s_j} > 0 \quad \Longrightarrow \quad \alpha_j > 0
$$

不管原始分数多小、多负，指数函数的输出永远是正数。这保证了每个位置的权重 ≥ 0——"关注程度"不会是负数。

### 性质 2：归一化

$$
\sum_{j=1}^{n} \alpha_j = \sum_{j=1}^{n} \frac{e^{s_j}}{\sum_k e^{s_k}} = 1
$$

所有权重加起来恰好等于 1，可以直接解读为**概率分布**或**信息分配比例**。"位置 $j$ 获得 30% 的注意力"比"位置 $j$ 的分数是 1.8"有意义得多。

### 性质 3：保序且放大差异

$$
s_i > s_j \quad \Longrightarrow \quad \alpha_i > \alpha_j
$$

分数大的位置权重一定更大（保序），但差距会被**指数函数放大**。原始分数差 1.0，经过 $e^x$ 后差距变成 $e \approx 2.718$ 倍。

关键观察：
- Δ = 0 时，所有位置权重相等（均匀分布）
- Δ 逐渐增大时，最高分位置越来越"独占"
- 但即使 Δ 很大，其他位置的权重**永远不会变成 0**——softmax 是"软"的

这就是 **soft**max 名字的由来：它**放大**差异，但**不完全消灭**低分位置，总是保留一点"后路"。

## 温度参数：控制 softmax 的"锐利度"

在实际应用中，有时需要调节 softmax 的尖锐程度。方法很简单——在分数上除以一个**温度参数** $T$：

$$
\alpha_j = \frac{e^{s_j / T}}{\sum_k e^{s_k / T}}
$$

| 温度 | 行为 | 类比 |
| --- | --- | --- |
| $T \to 0^+$ | 趋近 one-hot（hardmax） | "只看最相关的一个位置" |

| $T \to \infty$ | 趋近均匀分布 | "所有位置一视同仁" |

在 Attention 中，$\sqrt{d_k}$ 就起到了类似温度的作用——第 3 讲说过，除以 $\sqrt{d_k}$ 防止分数过大导致 softmax 退化成 one-hot。

**对应关系**：$\text{softmax}(QK^\top / \sqrt{d_k})$ 中的 $\sqrt{d_k}$ 就是温度 $T$。$d_k$ 越大，"温度"越高，分布越平滑。

## 为什么不用别的归一化方式

### 候选 1：直接除以总和

$$
\alpha_j = \frac{s_j}{\sum_k s_k}
$$

直接除以总和有两个致命问题：

1. **负分数怎么办？** $s = [3, -1, 2]$，总和 = 4，但 $\alpha_2 = -1/4 = -0.25$ 是负权重——含义不明
2. **总和为 0 怎么办？** $s = [1, -1]$，除以 0 直接崩溃

softmax 用 $e^x$ 把所有分数先映射到正数，从根源上避免了这两个问题。

### 候选 2：hardmax（只取最大值）

$$
\alpha_j = \begin{cases} 1 & \text{if } j = \arg\max_k s_k \\ 0 & \text{otherwise} \end{cases}
$$

hardmax 的问题第 2 讲已经讲过：不可微、丢信息。这里补充一个更根本的问题——**梯度为零**。hardmax 输出是阶跃函数，几乎处处梯度为 0，反向传播时什么也学不到。

### 候选 3：sparsemax

sparsemax 是 softmax 的"稀疏"变体：部分位置的权重会**恰好等于 0**（不只是接近 0）。

$$
\text{sparsemax}(\mathbf{s}) = \arg\min_{\mathbf{p} \in \Delta^n} \|\mathbf{p} - \mathbf{s}\|^2
$$

sparsemax 在一些场景下有优势（注意力更可解释），但标准 Transformer 不用它，因为：
- 计算比 softmax 复杂（需要排序）
- GPU 上缺乏高效实现
- 实验证明对 Transformer 性能提升不大

### 总结

| 方式 | 非负 | 归一化 | 可微 | 保留低分信息 | 计算效率 |
| --- | --- | --- | --- | --- | --- |
| 直接归一化 | ✗ | ✓ | ✓ | ✓ | 快 |
| hardmax | ✓ | ✓ | ✗ | ✗ | 快 |
| sparsemax | ✓ | ✓ | ✓ | 部分 | 慢 |
| **softmax** | **✓** | **✓** | **✓** | **✓** | **快** |

softmax 是唯一同时满足全部四个要求的函数。

## softmax 的梯度：为什么它能训练

softmax 可微这件事非常重要。它的 Jacobian 矩阵是：

$$
\frac{\partial \alpha_i}{\partial s_j} = \alpha_i (\delta_{ij} - \alpha_j)
$$

其中 $\delta_{ij}$ 是 Kronecker delta。这意味着：

- **$i = j$**（自身）：$\frac{\partial \alpha_i}{\partial s_i} = \alpha_i(1 - \alpha_i)$ — 权重越接近 0 或 1，梯度越小
- **$i \neq j$**（交叉）：$\frac{\partial \alpha_i}{\partial s_j} = -\alpha_i \alpha_j$ — 提高一个位置的分数会降低其他位置的权重

梯度消失的风险：当某个 $\alpha_i \to 1$（softmax 退化成 one-hot），自身梯度 $\alpha_i(1-\alpha_i) \to 0$。这就是为什么要除以 $\sqrt{d_k}$——**防止分数过大导致 softmax 过于尖锐，从而导致梯度消失**。

## 数值稳定性：一个工程细节

直接计算 $e^{s_j}$ 有溢出风险。如果 $s_j = 1000$，$e^{1000}$ 超出 float64 范围。

解决方法很简单——先减去最大值：

$$
\text{softmax}(s_j) = \frac{e^{s_j}}{\sum_k e^{s_k}} = \frac{e^{s_j - \max(s)}}{\sum_k e^{s_k - \max(s)}}
$$

## 完整的 PyTorch 实验

```python
import torch
import torch.nn.functional as F
import math

scores = torch.tensor([2.0, 1.0, 0.5, -1.0])

# ---- 手动实现 softmax ----
def manual_softmax(s):
    s_shifted = s - s.max()        # 数值稳定：减去最大值
    exps = torch.exp(s_shifted)    # e^(s - max)
    return exps / exps.sum()       # 归一化

print("手动 softmax:", manual_softmax(scores))
print("PyTorch softmax:", F.softmax(scores, dim=0))
# 结果完全一致

# ---- 性质验证 ----
alpha = F.softmax(scores, dim=0)
print(f"\n非负: 所有 α > 0? {(alpha > 0).all()}")         # True
print(f"归一化: Σα = {alpha.sum().item():.6f}")             # 1.000000
print(f"保序: scores 排序 = α 排序? "
      f"{(scores.argsort() == alpha.argsort()).all()}")       # True

# ---- 温度实验 ----
print("\n温度实验:")
for T in [0.1, 0.5, 1.0, 2.0, 5.0]:
    a = F.softmax(scores / T, dim=0)
    entropy = -(a * a.log()).sum().item()
    print(f"  T={T:.1f}: α={a.numpy().round(3)}, 熵={entropy:.3f}")

# ---- 梯度 ----
scores_g = scores.clone().requires_grad_(True)
alpha_g = F.softmax(scores_g, dim=0)
alpha_g[0].backward()  # 对第一个位置的权重求梯度
print(f"\n提高 s₁ 对各位置权重的影响:")
print(f"  ∂α/∂s = {scores_g.grad.numpy().round(4)}")
# 第一个位置正梯度（提高自己），其余负梯度（降低别人）

# ---- 数值稳定性 ----
big_scores = scores * 500  # 放大 500 倍
try:
    naive = torch.exp(big_scores) / torch.exp(big_scores).sum()
    print(f"\n朴素计算: {naive}")  # 可能出现 nan/inf
except:
    print("\n朴素计算: 溢出!")
print(f"稳定计算: {F.softmax(big_scores, dim=0)}")  # 正常
```

## 把 softmax 放回 Attention 里

现在我们可以完整理解注意力权重矩阵 $\alpha$ 的含义：

$$
\alpha_{ij} = \frac{e^{S_{ij}/\sqrt{d_k}}}{\sum_{k=1}^{n} e^{S_{ik}/\sqrt{d_k}}}
$$

**逐行阅读**这个矩阵：

- 第 $i$ 行：位置 $i$ 把注意力分配给所有位置的比例
- 每行之和 = 1（softmax 保证）
- 颜色越深 = 权重越大 = 从该位置取越多信息

softmax 把"分数"变成了"比例"——这就是为什么 Attention 的输出 $\alpha V$ 是一个**加权平均**，而不是某种奇怪的线性组合。

## 这一讲的三个核心结论

1. **softmax 满足注意力权重的所有要求**：非负、归一化、可微、保序。是唯一同时满足这四个条件且计算高效的函数。

2. **指数函数放大差异但不消灭低分**。分数高的位置获得更多权重，但其他位置永远不会完全归零——这是"soft"的核心含义。

3. **$\sqrt{d_k}$ 就是温度**。它控制 softmax 的锐利度：防止分数过大导致 one-hot 退化（梯度消失），也防止分数过小导致均匀分布（不区分重要性）。

  function softmax(arr, T) {
    T = T || 1;
    var mx = Math.max.apply(null, arr);
    var exps = arr.map(function(v) { return Math.exp((v - mx) / T); });
    var s = exps.reduce(function(a, b) { return a + b; });
    return exps.map(function(e) { return e / s; });
  }

  function initNorm() {
    var svg = document.getElementById('norm-svg');
    var infoEl = document.getElementById('norm-info');
    if (!svg) return;

    var scores = [2.5, -0.8, 1.5, 0.3];
    var labels = ['猫', '了', '垫子', '上'];
    var current = 'softmax';

    function compute(mode) {
      if (mode === 'softmax') return softmax(scores);
      if (mode === 'linear') {
        var s = scores.reduce(function(a, b) { return a + b; });
        return scores.map(function(v) { return v / s; });
      }
      if (mode === 'hardmax') {
        var mx = Math.max.apply(null, scores);
        return scores.map(function(v) { return v === mx ? 1 : 0; });
      }
      if (mode === 'sparsemax') {
        // Simplified sparsemax approximation
        var sorted = scores.slice().sort(function(a, b) { return b - a; });
        var cumSum = 0, k = 0;
        for (var i = 0; i < sorted.length; i++) {
          cumSum += sorted[i];
          if (sorted[i] > (cumSum - 1) / (i + 1)) k = i + 1;
        }
        cumSum = 0;
        for (var j = 0; j < k; j++) cumSum += sorted[j];
        var tau = (cumSum - 1) / k;
        return scores.map(function(v) { return Math.max(0, v - tau); });
      }
      return softmax(scores);
    }

    function draw() {
      var c = C();
      var alpha = compute(current);
      var maxA = Math.max.apply(null, alpha.map(Math.abs));
      var sx = 80, gapX = 140, barBase = 200, barMax = 120;

      var h = '';
      var modeNames = { softmax: 'softmax', linear: '直接归一化 s/Σs', hardmax: 'hardmax', sparsemax: 'sparsemax' };

      labels.forEach(function(l, i) {
        var x = sx + i * gapX;
        var a = alpha[i];
        var isNeg = a < 0;
        var barH = Math.max(2, (Math.abs(a) / (maxA + 0.001)) * barMax);
        var barY = isNeg ? barBase : barBase - barH;
        var barColor = isNeg ? c.danger : mColor;

        h += '<rect x="' + (x - 28) + '" y="' + barY + '" width="56" height="' + barH + '" rx="5" fill="' + barColor + '" opacity="0.7"/>';
        var valY = isNeg ? barBase + barH + 15 : barBase - barH - 6;
        h += '<text x="' + x + '" y="' + valY + '" text-anchor="middle" fill="' + (isNeg ? c.danger : c.text) + '" font-size="11" font-weight="700">' + (a >= 0 ? (a * 100).toFixed(1) + '%' : a.toFixed(3)) + '</text>';
        h += '<text x="' + x + '" y="' + (barBase + (isNeg ? barH + 32 : 18)) + '" text-anchor="middle" fill="' + c.text + '" font-size="11">' + l + '</text>';
        h += '<text x="' + x + '" y="' + (barBase + (isNeg ? barH + 47 : 33)) + '" text-anchor="middle" fill="' + c.muted + '" font-size="9" font-family="monospace">s=' + scores[i] + '</text>';

        // Warning for negative
        if (isNeg) {
          h += '<text x="' + x + '" y="' + (barY - 5) + '" text-anchor="middle" fill="' + c.danger + '" font-size="10" font-weight="700">⚠ 负权重!</text>';

        }
      });

      h += '<line x1="40" y1="' + barBase + '" x2="660" y2="' + barBase + '" stroke="' + c.border + '" stroke-width="1"/>';

      h += '<text x="640" y="76" text-anchor="end" fill="' + (allNonNeg ? c.success : c.danger) + '" font-size="10">全部≥0? ' + (allNonNeg ? '✓' : '✗') + '</text>';

      svg.innerHTML = h;

      var descs = {

        linear: '"了"的分数 -0.8，直接归一化后变成 <strong>负权重</strong>。语义不明："负关注度"是什么意思？而且如果分数总和恰好为 0，直接除以 0 崩溃。',

        // Stable exp

        h += '<text x="510" y="' + (y + 4) + '" text-anchor="middle" fill="' + c.success + '" font-size="11" font-weight="600">' + stableExp.toFixed(4) + '</text>';

      svg.innerHTML = h;

      var anyOverflow = baseScores.some(function(v) { return Math.exp(v * factor) === Infinity; });
      if (factor <= 1) {

      }
    }
